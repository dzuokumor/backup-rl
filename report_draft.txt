PROJECT OVERVIEW

This project implements a reinforcement learning agent for mitigating cascading failures in a power distribution grid modeled on the Lagos/EKEDC (Eko Electricity Distribution Company) network. The environment simulates a modified IEEE 14-bus system with 14 substations, 20 transmission lines, and 5 generators, where the agent must make real-time operational decisions to maintain grid stability. The core challenge is preventing cascading thermal overloads, where a single line failure can trigger a chain reaction of disconnections leading to a full blackout. Four RL algorithms (DQN, REINFORCE, PPO, and A2C) were trained and compared using the Stable Baselines 3 library, with PPO emerging as the best-performing method. The environment features DC power flow physics, stochastic fault injection, demand fluctuations, and a 3D nighttime Lagos cityscape visualization built with PyOpenGL and Pygame. An API endpoint and web-based GUI were developed to enable real-time monitoring and interaction with the trained agent.


ENVIRONMENT DESCRIPTION

Agent(s)

The agent represents an automated grid operator responsible for managing the EKEDC power distribution network. It observes the current state of the grid at each timestep and selects one of 53 discrete actions to maintain power delivery and prevent cascading failures. The agent operates in a single-agent setting, making sequential decisions every timestep (one step out of a 1000-step episode). It has the ability to shed or restore load at consumer buses, toggle transmission lines on or off, and increase or decrease generator output. The agent does not have direct control over random fault events or the underlying physics of power flow; it can only react to and preemptively manage the grid topology and generation dispatch. Its goal is to maximize the amount of load served while minimizing cascading failures over the duration of an episode.

Action Space

The action space is discrete with 53 possible actions:

- Action 0: Do nothing (idle). The agent chooses to take no action this timestep.
- Actions 1-11: Shed load at one of the 11 load buses. Each action reduces demand at the corresponding bus by 20% of its base load (LOAD_SHED_INCREMENT = 0.20). Load shedding fractions are capped at 100%.
- Actions 12-22: Restore load at one of the 11 load buses. Each action increases demand back by 20%, restoring previously shed load. Cannot restore below 0%.
- Actions 23-42: Toggle one of the 20 transmission lines. Switches the line between connected and disconnected states. Subject to a 5-step cooldown (LINE_SWITCH_COOLDOWN = 5) after each toggle to prevent rapid switching.
- Actions 43-47: Increase output of one of the 5 generators by 10% of its maximum capacity (GEN_REDISPATCH_INCREMENT = 0.10). Output changes are limited by the generator's ramp rate.
- Actions 48-52: Decrease output of one of the 5 generators by 10% of its maximum capacity. Output cannot drop below the generator's minimum output.

Invalid actions (e.g., restoring load that was not shed, toggling a line on cooldown, adjusting an unavailable generator) receive a penalty of -0.05 but do not crash the environment.

Observation Space

The observation is a 117-dimensional continuous vector (float32) that encodes the full observable state of the grid:

Per transmission line (20 lines x 3 = 60 values):
- Line loading ratio: current flow divided by thermal limit (0.0 to unbounded). Values above 1.0 indicate overload.
- Line status: binary indicator (1 = connected, 0 = disconnected).
- Overload timer: integer count of consecutive timesteps the line has been overloaded. After 3 consecutive steps (MAX_OVERLOAD_STEPS), the line auto-disconnects.

Per bus (14 buses x 3 = 42 values):
- Voltage: approximated bus voltage in per-unit (nominal = 1.0). Deviations indicate stress.
- Power injection: net power injection at the bus (positive = generation, negative = load).
- Load shed indicator: binary flag (1 if any load is being shed at this bus).

Per generator (5 generators x 2 = 10 values):
- Output fraction: current output divided by maximum capacity (0.0 to 1.0).
- Availability: binary indicator (1 = online, 0 = tripped/offline).

Global features (5 values):
- Total system demand (sum of all bus loads after demand multiplier).
- Total system generation (sum of all active generator outputs).
- Time of day (normalized 0.0 to 1.0, driving the sinusoidal demand cycle).
- Number of currently overloaded lines.
- Number of currently disconnected lines.

Reward Structure

The reward function is computed at each timestep as a weighted sum of six components:

R(t) = +2.0 * serve_ratio
       -1.0 * overload_penalty
       -0.5 * shed_ratio
       -0.3 * disconnect_ratio
       -10.0 * cascade_penalty
       -0.1 * (action != 0)
       -0.5 * voltage_penalty
       -action_penalty

Where:
- serve_ratio: fraction of total demand currently being served (range 0 to 1). This is the primary positive reward signal, incentivizing the agent to keep the lights on.
- overload_penalty: sum of excess loading beyond 1.0 for all active lines. Penalizes allowing lines to operate above thermal limits.
- shed_ratio: fraction of total demand that has been deliberately shed. Discourages excessive load shedding as a lazy solution.
- disconnect_ratio: fraction of total lines currently disconnected. Penalizes unnecessary line disconnections.
- cascade_penalty: 1.0 if any auto-disconnect (thermal protection trip) occurred this timestep, 0.0 otherwise. The -10.0 weight makes cascading failures the most heavily penalized event.
- action cost: -0.1 for any non-idle action, encouraging the agent to act only when necessary.
- voltage_penalty: sum of voltage deviations beyond the tolerance band (VOLTAGE_TOLERANCE = 0.05) across all buses.
- action_penalty: -0.05 if the chosen action was invalid (e.g., toggling a line on cooldown).

Terminal conditions:
- Blackout (terminated): load served drops below 30% of demand OR 8 or more lines are disconnected.
- Survived (truncated): agent successfully maintains the grid for 1000 timesteps.


SYSTEM ANALYSIS AND DESIGN

Deep Q-Network (DQN)

The DQN implementation uses the Stable Baselines 3 library with an MLP policy network. The architecture consists of fully-connected hidden layers (varying from [64,64] to [256,256,256] across hyperparameter runs) with ReLU activations. The network maps the 117-dimensional observation to Q-values for each of the 53 actions. Standard DQN features are employed: an experience replay buffer (50,000-100,000 transitions), a target network updated at fixed intervals (500-2000 steps), and an epsilon-greedy exploration strategy that decays linearly from 1.0 to a final epsilon over a configurable fraction of training. The replay buffer enables off-policy learning by storing and sampling past transitions uniformly, breaking temporal correlations. The target network provides stable Q-value targets during training, updated periodically by copying the online network weights. Training was conducted over 100,000 timesteps per hyperparameter configuration, with the best configuration then trained for 1,000,000 timesteps.

Policy Gradient Methods

REINFORCE: A vanilla policy gradient implementation using a custom PolicyNetwork (MLP with configurable hidden layers and softmax output). The network parameterizes a categorical distribution over 53 actions. After each complete episode, Monte Carlo returns are computed with configurable discount factor gamma, normalized (zero mean, unit variance), and used to compute the policy gradient loss. Gradient clipping (max_grad_norm) prevents training instability. Unlike DQN, REINFORCE is on-policy and updates only from complete episode trajectories, which makes it less sample efficient but provides unbiased gradient estimates.

PPO (Proximal Policy Optimization): Implemented via Stable Baselines 3 using an MLP policy with separate value and policy heads. PPO uses a clipped surrogate objective to constrain policy updates, preventing destructively large steps. Key features include Generalized Advantage Estimation (GAE) with configurable lambda for variance reduction, mini-batch updates over collected rollouts, and entropy regularization to maintain exploration. The clip range, number of epochs per update, and batch size were all tuned across 10 runs.

A2C (Advantage Actor-Critic): Also implemented via Stable Baselines 3 with shared feature extraction layers feeding into separate policy and value heads. A2C performs synchronous updates using n-step returns (configurable n_steps from 16 to 64). The value function coefficient (vf_coef) balances policy and value losses, while entropy regularization (ent_coef) prevents premature convergence. Gradient clipping (max_grad_norm) stabilizes training.

[INSERT: results/plots/convergence_comparison.png]

Convergence speed varies substantially across methods. PPO converges to a stable high-reward policy within approximately 3,000-5,000 episodes at 100k steps (with continued improvement at 1M steps). Generalization

In contrast, the PPO agent learned strategies such as proactive line switching and selective load shedding that allow it to survive entire episodes and maintain high load delivery.


CONCLUSION AND DISCUSSION

PPO emerged as the best-performing algorithm for the power grid cascading failure mitigation task, achieving a final 100-episode mean reward of 211.60 at 1M training steps, significantly outperforming DQN (158.62), REINFORCE (62.10), and A2C (32.16).

PPO's superiority is attributable to several factors specific to this environment. The clipped surrogate objective provides stable policy updates in a high-dimensional action space (53 discrete actions), preventing the policy collapse observed in some A2C and REINFORCE runs. The use of GAE for advantage estimation effectively balances bias and variance when estimating the value of actions whose consequences (cascading failures) may not manifest for several timesteps. PPO's mini-batch update scheme also provides computational efficiency during training.

DQN performed as the second-best method, demonstrating that value-based approaches remain competitive for discrete action spaces. However, DQN's performance was limited by the challenge of accurately estimating Q-values for 53 actions from a 117-dimensional observation space, and by the inherent difficulty of off-policy learning in a highly stochastic environment where the optimal action depends heavily on the exact grid configuration.

REINFORCE, despite its theoretical simplicity, struggled with sample efficiency. The high variance of Monte Carlo returns in an environment where episode lengths range from 9 to 1000 steps made gradient estimates unreliable. The algorithm required significantly more training to approach the performance of PPO or DQN.

A2C showed promising results in the 100k sweep (best run achieving 95.18 mean reward) but failed to scale effectively to 1M steps, suggesting sensitivity to hyperparameter choices over longer training horizons. The synchronous update scheme may have contributed to suboptimal exploration in later stages of training.

Key agent behaviors observed during evaluation include:
1. Proactive line switching: The PPO agent learned to disconnect transmission lines preemptively to redistribute power flow and prevent thermal overloads.
2. Selective load shedding: When faults occur, the agent sheds load at lower-priority buses to maintain overall grid stability.
3. Generator redispatch: The agent adjusts generator outputs to balance supply and demand across different grid configurations.

Limitations and future work:
- The current model uses a fixed policy that cannot adapt to novel states during deployment (e.g., permanent generator loss). Online learning or meta-RL could address this.
- The DC power flow approximation simplifies the true AC power flow dynamics. Extending to full AC power flow would increase realism.
- The agent was trained on a single grid topology. Transfer learning across different grid configurations would improve practical applicability.
- Integration with the DATMA (Dependency-Aware Temporal Memory Architecture) capstone project could provide the agent with temporal context for better anticipation of cascading events.
